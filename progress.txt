## Codebase Patterns
- Use `createApp()` from `src/server/app.ts` to get a testable Express app (no `listen` call)
- `src/server/index.ts` is the thin entry point that imports `createApp()` and calls `listen()`
- MCP endpoint at POST /mcp creates a fresh server+transport per request (no session persistence across requests)
- MCP Streamable HTTP transport requires `Accept: application/json, text/event-stream` header
- MCP responses come as SSE (text/event-stream) — parse `data:` lines to extract JSON-RPC payloads
- `createMcpExpressApp()` from the SDK adds body-parsing middleware that conflicts with `express.json()` — use plain `express()` instead when manual MCP handling is in place
- Tests use Node built-in test runner (`node:test`) with `tsx` for TypeScript — run via `npm test`
- Without DATABASE_URL, the repository falls back to in-memory seed data (good for tests)
